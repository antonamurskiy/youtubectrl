# Vids — native iOS rewrite

Pure SwiftUI + UIKit replacement for the React/Capacitor shell. Talks to the
existing `server.js` over the same HTTP + WS endpoints. The server is unchanged.

## Status

**Phase 1 of 10 complete** — project scaffold, WS connection, store wiring,
RootView shell, header + tabs, feed list (UICollectionView + diffable data
source), now-playing bar (CADisplayLink-driven scrubber), placeholder
terminal view, push category registration. Builds clean for iOS Simulator.

## Build

The project is generated by [xcodegen](https://github.com/yonaskolb/XcodeGen).
After editing `project.yml` or adding/removing source files, regenerate:

```bash
cd ios-native && xcodegen
```

Simulator build (no signing):
```bash
xcodebuild -project Vids.xcodeproj -scheme Vids -configuration Debug \
  -destination "generic/platform=iOS Simulator" build
```

Device build:
```bash
DEVICE=00008150-001241D11EF2401C
xcodebuild -project Vids.xcodeproj -scheme Vids -configuration Debug \
  -destination "id=$DEVICE" -allowProvisioningUpdates build
```

First device build needs to provision the new bundle ID
`com.antonamurskiy.vids`. If `-allowProvisioningUpdates` fails from a
non-interactive shell, open `Vids.xcodeproj` in Xcode once, click
**Signing & Capabilities → Team → Anton Amurskiy**, and let Xcode register
the App ID. After that the CLI flow works.

## Layout

```
Vids/
  App/                  VidsApp + AppDelegate + ServiceContainer (DI root)
  Networking/           ApiClient (async/await) + WSClient (URLSessionWebSocketTask) + Codable types
  Stores/               @Observable stores (Playback, Feed, Terminal, Theme, Push)
  Views/
    RootView.swift      Top-level layout + dismount-on-terminal feed gating
    HeaderView.swift    Tabs + search + status dots
    Feed/               UICollectionView host + cell views + ThumbnailCache
    NowPlaying/         CADisplayLink scrubber + bar
    Terminal/           Tmux tabs + (phase 7) SwiftTerm host
  Push/                 UNUserNotificationCenter delegate (phase 8)
  Player/               (phase 5/6) AVPlayerHost, LiveSyncEngine
  Native/               (phase 5/8) NowPlayingCenter, VolumeButtons, AirPlayPicker
```

## Things deliberately NOT done in phase 1

- **Inline AVPlayer + PiP**: phase 5 ports `NativePlayerPlugin.swift` (~1000
  LOC) — most of the file carries over with refactor.
- **Live sync**: phase 6 ports the PDT anchor + self-calibrating bias loop
  from `PhonePlayer.jsx`.
- **SwiftTerm**: phase 7. Add as SPM dependency, wire to `/ws/tmux`.
- **APNs token registration**: phase 8 POSTs to `/api/apns-register`.
- **Hover preview videos in cells**: deliberately deferred. The React app
  was buffering 5+ silent streams for cards in the prefetch band; the new
  app will use a single shared muted `AVPlayer` swapped via `replaceCurrentItem`,
  but that needs the AVPlayerHost from phase 5 first.

## Server contract

Identical to the React app. See `server.js` in repo root. WS message types:
`playback`, `tmux`, `claude-feed`, `claude`, `pong`. HTTP endpoints:
`/api/play`, `/api/seek`, `/api/skip`, `/api/playpause`, `/api/stop`,
`/api/home`, `/api/search`, `/api/live`, `/api/history`, `/api/preview-url`,
`/api/tmux-select`, `/api/tmux-rename`, `/api/tmux-color`, `/api/tmux-send`.
